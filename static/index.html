Active Deduplication Approach — AI-Flavored Pipeline (Tiered Funnel)

Active Algorithm
Pipeline Workflow
1. Pair Screening

Indexes reports into demographic, drug product, and adverse event datasets. Candidate duplicate pairs must share at least one Product Active Ingredient (PAI) and one Adverse Event (AE), with report dates within 365 days.

2. Pairwise Probabilistic Comparison (Tier 1)

Computes a record-linkage weight score over structured fields (age, sex, country, date, drugs, events) and narrative features. If score is ≥ Duplicate Threshold, they are classed as duplicates; if ≥ Rescue Threshold, they go to Tier 2.

3. Narrative LLM Adjudicator (Tier 2)

Ambiguous pairs falling in the rescue window are passed to the AI to compare case narratives in parallel. The AI uses the prompt configuration to decide if the cases describe the same event.

4. Modularity-Based Grouping & Selection

Forms groups using Clauset–Newman–Moore community detection on duplicate edges and selects the optimal reference case.

Active Deduplication Approach — InfoViP paper pipeline (deterministic, no AI)

      Active Algorithm
    

    
    

      

        
Pipeline Workflow

        

          

            
1. Pair Screening

            
Indexes reports into demographic, drug product, and adverse event datasets. Candidate duplicate pairs must share at least one Product Active Ingredient (PAI) and one Adverse Event (AE), with report dates within 365 days.

          

          
          

            
2. Pairwise Probabilistic Comparison

            
Computes a record-linkage weight score over structured fields (age, sex, country, date, drugs, events) and regex-extracted narrative features (diagnosis, medical history, family history, symptoms, drug mentions, dates).

          

          
          

            
3. Modularity-Based Grouping

            
Forms a network graph from duplicate pairs. Connected components are recursively split using Clauset–Newman–Moore community detection unless edge density meets specified group-size density exemptions.

          

          
          

            
4. Reference Case Selection

            
For each group, selects the report with the largest sum of connecting edge weights. Ties are broken by suspect products + AE count, narrative completeness, and most recent receive date.

Bootstrap compare macro-F1

    
Paired bootstrap over the benchmark series (resample series with replacement, recompute macro-F1 each time) — gives a 95% CI and p-value on the difference between two conditions, instead of just a single-run point estimate. Only series with a scored, non-hidden run under both conditions are used.
